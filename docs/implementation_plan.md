# План реализации - Оптимизация N+1 запросов

## Обзор
Текущая реализация `main/views.py` выполняет отдельный SQL-запрос для получения изображений каждого товара. Это приводит к серьезному падению производительности ("N+1 проблема"). Мы исправим это, загружая все необходимые данные за 2 запроса (1 для товаров + 1 для картинок).

## Предлагаемые изменения

### Представления Main (`main/views.py`)
#### [MODIFY] [views.py](file:///c:/Users/rober/Desktop/SITE_MARO/maro/main/views.py)
- Использовать `Product.objects.prefetch_related('images').all()` для предварительной загрузки изображений.
- Удалить цикл в `index`, который вручную присваивал `product.extra_images_list`, так как теперь мы будем использовать связь напрямую или через существующий метод, который подхватит кэш.

### Шаблоны (`products/index.html`)
- Убедиться, что шаблон использует связь эффективно. В данный момент он вызывает `product.get_extra_images`, который делает `self.images.all()`. Благодаря `prefetch_related`, этот вызов не будет делать запрос к БД, а возьмет данные из кэша Django.
